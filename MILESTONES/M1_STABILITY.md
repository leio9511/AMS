## 🔧 技术方案 (TL)

### 根因分析

**核心问题**: `send_msg` 使用 `subprocess.run(["openclaw", "message", "send", ...])`

**问题链**:
1. 每次发送消息启动新 Node.js 进程
2. 加载所有 OpenClaw 插件 (adp, qqbot, wecom, ddingtalk)
3. 资源消耗大 → 进程可能被系统杀死
4. 日志被插件信息淹没 → 无法排查

### 解决方案

**推荐方案**: 使用 Gateway WebSocket API

```python
def send_msg(text):
    """通过 Gateway WebSocket 发送消息"""
    request = {
        "type": "req",
        "id": str(uuid.uuid4()),
        "method": "chat.send",
        "params": {"channel": "telegram", "target": TARGET, "message": text}
    }
    ws.send(json.dumps(request))
```

**优点**: 无需启动新进程，资源消耗低，日志干净

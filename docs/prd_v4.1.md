## 1. 产品目标

打造一套全自动化的套利监控系统，能够：
1. **实时监控** ETF 和可转债的折溢价异动
2. **智能诊断** 折价品种的"预期下修"风险
3. **自动产出** 结构化分析报告

## 2. 核心功能规格

### 2.1 报告必须包含实时数据

报告需包含以下实时数据字段：

| 字段 | 说明 | 数据源 |
|------|------|--------|
| ETF代码 | 如 sh513330 | 腾讯行情API |
| ETF名称 | 如 恒生科技ETF | 腾讯行情API |
| 现价 (ask1) | 卖一价 | 腾讯行情API |
| IOPV | 实时净值 | 腾讯行情API |
| 折溢价率 | (ask1/IOPV - 1) × 100% | 计算得出 |
| 更新时间 | 北京时间 | 系统时间 |

**实时性要求**：
- 盘中每30秒刷新一次
- 开盘时刻(09:26-09:30)必须播报

### 2.2 折价品种"预期下修"智能诊断逻辑

#### 2.2.1 触发条件

当检测到以下情况时，激活智能诊断：
- ETF 折价率 > 0.8% (diff < -0.008)
- 且 ETF 属于 `STALE_NAV_CODES` 列表（底层资产为美股/海外资产）

#### 2.2.2 诊断逻辑

```
IF ETF底层资产 == 美股指数 AND 当前时间 == A股开盘时段(09:30-10:00):
    检查隔夜美股走势:
    - IF 美股隔夜下跌 > 1%:
        → 诊断结论: "折价反映隔夜美股下跌预期，非纯套利机会"
        → 建议操作: "观望，等待净值更新后重新评估"
    - IF 美股隔夜上涨:
        → 诊断结论: "异常折价，可能存在套利窗口"
        → 建议操作: "关注申购通道开放情况"
    - IF 美股休市:
        → 诊断结论: "使用前一日净值估算，存在时滞风险"
        → 建议操作: "谨慎操作"
ELSE:
    → 标准诊断: "常规折溢价波动"
```

#### 2.2.3 美股走势参考数据源

| 指数 | 代码 | 数据源 |
|------|------|--------|
| 纳斯达克100 | NDX | 新浪财经/东方财富 |
| 标普500 | SPX | 新浪财经/东方财富 |
| 道琼斯 | DJI | 新浪财经/东方财富 |

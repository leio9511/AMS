# PR-002: Create AMS Architecture BEST_PRACTICES Documentation

## 1. Objective
Generate documentation at the root of the AMS project outlining the correct directory structure and import standards to prevent future structural chaos.

## 2. Scope & Implementation Details
- **Create `AMS/BEST_PRACTICES.md`**: Add a new markdown file at the root.
- **Document Directory Structure**: Explicitly define the purpose of the `scripts/` directory (for core executable code and modules) and the `tests/` directory (for pytest files).
- **Document Import Standards**: Provide clear examples of how to import internal modules (e.g., `from scripts.qmt_client import QMTClient`).
- **Define Root Rule**: Add a strict rule stating that no new Python implementation files should be dumped in the root `AMS/` directory without explicit justification.

## 3. TDD & Acceptance Criteria
- `AMS/BEST_PRACTICES.md` exists.
- The documentation clearly explains the `scripts/` vs `tests/` division.
- The documentation includes correct import syntax examples.
- The documentation includes the strict rule against polluting the project root.
